## Project Overview
This project predicts **customer churn** in the telecommunications industry.  
The goal is to combine **machine learning accuracy** with **business insights** that help reduce churn and improve retention.

Designed to showcase:
-  Technical skills in Python, ML, and visualization.  
-  Ability to translate data into **business solutions**.  
-  Clean, organized, and recruiter-friendly portfolio.

---

##  Dataset
- **Source:** Telco Customer Churn dataset (Kaggle, 2025).  
- **Size:** 7,043 customers, multiple features (contract type, monthly income, monthly bill, internet usage, support tickets, feedback sentiment).  
- **Target:** `Churn` (customer left or stayed).  

---

##  Methodology
1. **EDA** → distributions, churn rates, correlations.  
2. **Data Prep** → encoding, train/test split.  
3. **Models** → Logistic Regression, Decision Tree, Random Forest.  
4. **Evaluation** → accuracy, precision, recall, F1-score.  
5. **Feature Importance** → ranking variables driving churn.  

---

##  Results
| Model               | Accuracy |
|---------------------|----------|
| Logistic Regression | 91%      |
| Decision Tree       | 90%      |
| Random Forest       | 96% ⭐   |

- **Random Forest** → ⭐ **96% accuracy** (best model).  
- **Key insight:**  
  -  Support tickets = most influential (≈0.51).  
  -  Contract type and  billing also matter.  

---

##  Conclusions & Recommendations
====================

- **Performance:** Random Forest is the most reliable model.  
- **Business implications:** Churn is strongly linked to support interactions and contract type.  
- **Actions:**
  - Monitor customers with multiple support tickets → priority assistance.  
  -  Encourage long-term contracts → discounts & loyalty benefits.  
  -  Flexible billing plans → reduce churn for price-sensitive customers.  
  -  Improve support experience → prevent dissatisfaction from escalating.
